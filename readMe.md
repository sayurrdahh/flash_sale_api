1. Entity (도메인 모델) 설계
   가장 먼저 데이터베이스 테이블과 1:1로 매핑될 객체를 만듭니다. 시스템의 근간이 되는 데이터 구조를 정의하는 단계입니다.

작업 내용: Coupon, CouponIssue (발급 내역) 클래스 생성.

핵심 포인트: 어떤 필드가 필요한지(수량, 상태 등), 엔티티 간의 연관 관계(1:N 등)는 어떻게 맺을지 정의합니다. JPA를 사용한다면 @Entity, @Id, @Column 등의 어노테이션을 세팅합니다.

2. Repository (데이터 접근 계층) 구현
   엔티티를 데이터베이스에 저장, 수정, 조회, 삭제(CRUD)하기 위한 인터페이스를 만듭니다.

작업 내용: CouponRepository, CouponIssueRepository 생성.

핵심 포인트: Spring Data JPA를 활용해 기본 쿼리 메서드를 만들고, 추후 동시성 제어를 위한 비관적 락(Pessimistic Lock) 쿼리 등도 이 계층에 정의하게 됩니다.

3. Service (비즈니스 로직) 구현 ★가장 중요★
   프로젝트의 핵심 로직이 들어가는 곳입니다. 트래픽 처리와 데이터 정합성 유지라는 역량을 가장 잘 보여줄 수 있는 계층입니다.

작업 내용: CouponService 클래스 생성.

핵심 포인트: * 쿠폰 잔여 수량 확인 및 차감 로직.

중복 발급 방지 로직.

@Transactional을 통한 트랜잭션 관리.

(추후 고도화 단계에서) Redis 캐시 조회, 분산 락 적용, 메시지 큐 발행 로직 등이 모두 이곳에 추가됩니다.

4. Test Code (테스트 코드) 작성
   일반적인 CRUD 애플리케이션이라면 Controller 구현 후 테스트를 하기도 하지만, 동시성 프로젝트에서는 Service 계층이 완성된 직후에 반드시 멀티스레드 환경의 테스트를 작성해야 합니다.

작업 내용: CouponServiceTest 작성.

핵심 포인트: 100개의 쿠폰에 1,000명의 유저가 동시에 접근했을 때 발생하는 문제(초과 발급)를 코드로 증명하고, 이를 해결해 나가는 과정을 확인합니다.

5. Controller (프레젠테이션 계층) & DTO 구현
   외부(클라이언트나 프론트엔드)로부터 HTTP 요청을 받고 응답을 내려주는 역할을 합니다.

작업 내용: CouponController 클래스와 클라이언트 간 데이터를 주고받을 CouponRequestDTO, CouponResponseDTO 생성.

핵심 포인트: API URI 설계 (예: POST /api/v1/coupons/issue), 클라이언트가 보낸 데이터의 유효성 검증(@Valid), Service에서 처리된 결과를 DTO로 변환하여 반환합니다.

6. Exception Handling (글로벌 예외 처리)
   "쿠폰이 모두 소진되었습니다" 또는 "이미 발급받은 쿠폰입니다"와 같은 예외 상황을 깔끔하게 처리합니다.

작업 내용: @RestControllerAdvice를 활용한 글로벌 예외 핸들러 작성.

핵심 포인트: 클라이언트에게 표준화된 에러 응답(HTTP 상태 코드 및 에러 메시지)을 내려주도록 공통화합니다.
